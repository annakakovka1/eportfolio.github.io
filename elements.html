AUC Analysis: Logistic Regression

For the classification experiment, logistic regression was applied to a standard binary classification dataset. The main parameter varied was C, which controls the strength of regularisation. Smaller values impose stronger regularisation, while larger values allow the model to fit the data more freely.

from sklearn.linear_model import LogisticRegression
from sklearn.metrics import roc_auc_score

for c in [0.01, 0.1, 1, 10, 100]:
    model = LogisticRegression(C=c, max_iter=5000)
    model.fit(X_train, y_train)
    y_prob = model.predict_proba(X_test)[:, 1]
    auc = roc_auc_score(y_test, y_prob)
    print(c, auc)

AUC Results

Parameter (C) Test AUC Interpretation
0.01 0.9799 Good performance, but likely too constrained by strong regularisation.
0.10 0.9906 Improved discrimination as the model became more flexible.
1 0.9958 Best observed AUC, suggesting an effective balance between fit and generalisation.
10 0.9954 Very similar to the best result, showing little additional benefit.
100 0.9939 Slight drop, suggesting diminishing returns and possible overfitting.

Key AUC Finding

The best AUC was achieved at C = 1. Increasing model flexibility improved AUC up to a point, but very large values did not produce better classification and led to a slight decline. This shows that stronger complexity does not always lead to stronger generalisation.

R² Analysis: Ridge Regression

For the regression experiment, Ridge regression was used to evaluate the impact of the alpha parameter on model fit. Here, alpha controls the strength of L2 regularisation. Small alpha values apply light regularisation, while large alpha values strongly constrain the model and can lead to underfitting.

from sklearn.linear_model import Ridge
from sklearn.metrics import r2_score, mean_squared_error
import numpy as np

for alpha in [0.01, 0.1, 1, 10, 100]:
    model = Ridge(alpha=alpha)
    model.fit(X_train, y_train)
    pred = model.predict(X_test)
    r2 = r2_score(y_test, pred)
    rmse = np.sqrt(mean_squared_error(y_test, pred))
    print(alpha, r2, rmse)

R² Results

Parameter (alpha) Test R² RMSE Interpretation
0.01 0.4871 53.26 Reasonably strong fit with very light regularisation.
0.10 0.4918 53.01 Best observed result, indicating a useful balance between stability and fit.
1 0.4384 55.73 Performance began to decline as regularisation increased.
10 0.1564 68.30 Strong reduction in explanatory power, indicating underfitting.
100 0.0091 74.02 Very poor fit, showing that excessive regularisation harms prediction quality.

Key R² Finding

The best R² result was observed at alpha = 0.1. As alpha increased further, R² decreased sharply and RMSE worsened, showing a clear underfitting pattern. This demonstrates that regularisation must be tuned carefully because too much constraint can remove useful predictive structure from the model.

Critical Reflection

Strength of the Activity

This task was valuable because it moved beyond simply reading evaluation metrics and required active experimentation. Changing parameters made it easier to see that model performance is shaped by design choices, not only by the dataset itself.

Limitations of the Original Notebook

The original notebook functioned more as a tutorial than a full analytical investigation. In particular, the R² example used fixed values rather than a trainable model, which meant that parameter tuning had to be added in order to make the task meaningful.

Interpretation of AUC

The AUC experiment showed that classification performance can improve when regularisation is relaxed, but only up to an optimal point. This reinforced the importance of balancing flexibility with generalisation rather than assuming that a more complex model is automatically better.

Interpretation of R²

The regression experiment clearly demonstrated that R² is highly sensitive to regularisation choices. Excessive regularisation reduced predictive quality and increased error, illustrating how underfitting can emerge when the model is made too rigid.

Professional Insight

This activity reinforced that model evaluation is not just about obtaining a number. It also involves understanding the conditions under which a metric is produced, whether the experiment is valid, and whether the performance is likely to generalise beyond the current data split.

Further Improvement

If this work were extended, I would use cross-validation rather than a single train/test split and compare additional models. This would provide a more robust basis for deciding whether performance changes are stable or merely sample-specific.

Overall, this task strengthened my understanding of model evaluation by showing that performance metrics such as AUC and R² are only meaningful when interpreted alongside model design, parameter tuning and methodological rigour. It highlighted the importance of critical reflection in machine learning practice and showed that effective evaluation depends on both technical execution and sound analytical judgement.
